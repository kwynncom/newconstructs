Python Engineer Code Exercise

Instructions

  1. Once you receive these instructions, we'd like you to deliver your solution or partial solution within 24 hours.
  2. Retrieve the html SEC filing from  http://www.sec.gov/Archives/edgar/data/9092/000000909214000004/bmi-20131231x10k.htm
  3. Please produce the following, to two separate txt files:
    1. A text version of the filing (at least partially formatted) [ document.txt ]
    2. A list of paragraphs, excluding text in tables, from the filing that contain a dollar sign, plus the paragraph start index and the paragraph index in the text document from step A. [ paragraphs.txt ]

Deliverables

  1. Source code (excluding libraries)
  2. README containing
    1. Brief description of your approach to the problem
    2. Approximate amount of time you spent working on your solution
    3. List of any libraries used, including version information
    4. Instructions for running
    5. Location of output files (document.txt and paragraphs.txt)

Submission

  1. Please submit your solution by sharing a git repo containing the deliverables listed above with us via either github or bitbucket.
  2. You can share it via github by following the instructions here and adding username [removed].
  3. You can share it via bitbucket by following the instructions in this video and sharing with username [removed].
  4. Please also send an email to [removed] to let him know you've submitted your solution.

Grading (not in order of importance)

  1. Code style
  2. Leveraging of libraries
  3. Correctness of solution
  4. Generality of solution (we will apply this solution to other sec filings and see how well it performs)
  5. Run speed of solution

Notes

  1. We expect you to spend 2-4 hours on this project. If you get to the time limit and have a partially working solution, please submit it. We want to get an idea of how you approach the problem, and completion isn't necessarily a requirement for that.
  2. Conversion of html version of the filing to a text version of the filing is required.
  3. Your solution must be runnable in Python3 (we are using Python 3.4)
  4. This test simulates an in-house use case, so we will compare your solution to ours.
